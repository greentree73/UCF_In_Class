## 💡 What are Resolvers?

**Resolvers are functions that fetch data for GraphQL fields.** They are the bridge between your GraphQL schema and your data sources.

Think of resolvers like **waiters in a restaurant**:
- 📋 **Schema** = The menu (what's available)
- 🍽️ **Query** = Customer's order (what they want)
- 👨‍🍳 **Resolver** = Waiter who gets the food from the kitchen (fetches the data)

```graphql
# When a client asks for this...
query {
  posts {
    title
    author
    isPopular  # ← Computed field!
  }
}
# ...resolvers handle the actual data fetching and computation
```

## 🧠 Why we use Resolvers

- **Flexible Data Sources:** Connect to databases, APIs, files, or any data source
- **Data Transformation:** Compute, format, and derive new data on-demand
- **Lazy Loading:** Only fetch data when fields are actually requested
- **Business Logic:** Apply rules and calculations at the field level
- **Error Handling:** Control how errors are handled and returned
- **Performance:** Optimize queries and implement caching strategies

## 🧪 Example 1: Query Resolvers (Database Fetching)

**Query resolvers** fetch data from databases or other sources.

```graphql
query {
  posts {
    id
    title
    author
    views
  }
}
```

**Resolver Implementation:**
```typescript
Query: {
  posts: async () => {
    // This resolver fetches from MongoDB
    return await Post.find().sort({ createdAt: -1 });
  }
}
```

**What happens:**
1. Client requests `posts`
2. GraphQL calls the `posts` resolver
3. Resolver queries MongoDB
4. Returns array of post objects

## 🧪 Example 2: Field Resolvers (Computed Fields)

**Field resolvers** compute derived data or transform existing data.

```graphql
query {
  posts {
    title
    views
    isPopular    # ← Computed by resolver!
    summary      # ← Generated by resolver!
  }
}
```

**Resolver Implementation:**
```typescript
Post: {
  isPopular: (post) => {
    return post.views >= 30; // Business logic!
  },
  
  summary: (post) => {
    return post.content.length <= 100 
      ? post.content 
      : post.content.substring(0, 100) + '...';
  }
}
```

**What happens:**
1. `posts` resolver returns raw post data
2. For each post, GraphQL calls field resolvers
3. `isPopular` computes popularity based on views
4. `summary` creates truncated content preview

## 🔍 Resolver Types Explained

### 1. **Query Resolvers**
- Top-level entry points
- Fetch data from databases/APIs
- Handle arguments and filtering

### 2. **Field Resolvers** 
- Transform individual fields
- Add computed properties
- Format dates, strings, numbers

### 3. **Type Resolvers** (Advanced)
- Resolve abstract types
- Handle unions and interfaces

```graphql
query {
  posts {
    title      # Only title and isPopular resolvers run
    isPopular  # Other field resolvers are skipped!
  }
}
```
