# 🚀 Apollo MERN Lab (TypeScript Backend)

Practice reviewing and running a full-stack Apollo + MERN Exercise where the backend is implemented in TypeScript.

**Estimated time:** 7-10 minutes

## 🎯 Goal

Identify how the frontend and TypeScript backend are connected, and run the code from either Starter or Solution.

## 📁 Folders

- **Starter/**: Guided version with discussion prompts
- **Solution/**: Completed reference version

## 🧭 Choose Your Path

### Starter Path

1. Open `Starter/README.md`
2. Install dependencies:
   ```bash
   npm run install:all
   ```
3. Start both apps:
   ```bash
   npm run dev
   ```
4. Optional: seed data
   ```bash
   npm run seed
   ```

### Solution Path

1. Open `Solution/README.md`
2. Install dependencies:
   ```bash
   npm run install:all
   ```
3. Start both apps:
   ```bash
   npm run dev
   ```
4. Optional: seed data
   ```bash
   npm run seed
   ```

## 🧪 TypeScript Backend Commands

Run these inside either `Starter/backend` or `Solution/backend`:

```bash
npm run dev      # TS watch mode with tsx
npm run start    # run server.ts
npm run seed     # run seedData.ts
npm run build    # Type check (tsc --noEmit)
```

## ✅ Results

By the end of this Lab, students can:

- Navigate Starter vs Solution quickly
- Run a TypeScript backend with Apollo Server
- Explain the role of core configuration files and startup scripts
