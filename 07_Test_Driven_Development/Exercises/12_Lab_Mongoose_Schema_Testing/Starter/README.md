# Lab: Mongoose Schema Testing (Starter)

## 🎯 Objective
Write **three Cypress tests** that verify:
- Mongoose schema validation rules
- Mongoose middleware behavior

You will complete the TODO tests in `cypress/e2e/mongoose-schema.cy.ts`.

## ⏱️ Estimated Time
25-35 minutes

## 🧠 What You Are Testing
This project has a `Book` model with:
- `title` required and minimum length 3
- `author` required
- `slug` generated by Mongoose middleware (`pre("validate")`)

Your tests should prove those rules actually work through the GraphQL API.

## 🚀 Setup
1. Install dependencies:
   ```bash
   npm install
   ```
2. Create `.env` file:
   ```bash
   cp .env.example .env
   ```
3. Start MongoDB locally.
4. Start the API server:
   ```bash
   npm run dev
   ```

## 📝 Your Task: Create 3 Tests
Open:
- `cypress/e2e/mongoose-schema.cy.ts`

Replace each TODO with a working test.

### Test 1 (Valid Input + Middleware)
Write a test that:
- sends a valid `createBook` mutation,
- expects success (no GraphQL errors),
- checks returned `slug` matches the generated format.

### Test 2 (Validation: short title)
Write a test that:
- sends `title` with fewer than 3 characters,
- expects GraphQL `errors` to exist,
- confirms message includes `validation failed`.

### Test 3 (Validation: missing author)
Write a test that:
- sends input with missing/empty `author`,
- expects GraphQL `errors` to exist,
- confirms schema `required` behavior is enforced.

## ▶️ Run Commands
- Run Cypress in UI:
  ```bash
  npm run cypress:open
  ```
- Run only this spec (headed):
  ```bash
  npm run cypress:run:headed -- --spec cypress/e2e/mongoose-schema.cy.ts
  ```

## ✅ Success Criteria
- [ ] All 3 tests are implemented (no TODO errors).
- [ ] All 3 tests pass.
- [ ] You verified one middleware behavior and two validation failures.
